PRE-BOOTCAMP CYBERSECURITY

The PRE-BOOTCAMP CYBERSECURITY web page.

My criticism of Command Line Overview  and its follow on pages include but are not limited to:

  • Mac OS X appears to be ignored.
  • There ought to be a table with 4 columns: function (what you are trying to do), command in Windows power shell, command in linux, command in MAC
  • Pointers to tutorials for all 3 ecosystems.nwet
  • In the linux page, ipconfig has been deprecated for years.  Use the ip command.  Netstat has been deprecated.  Use ss.  traceroute using ICMP, but tracepath uses UDP.  A lot of network admins block ICMP at the border gateway because of the infamous IPv4 "ping of death".

My criticism of Basic Networking and its following pages is:

  • The graphics on the Basic Networking page are broken.  The source of one of the graphics is https://s3.amazonaws.com/General_V88/boomyeah2015/codingdojo/curriculum/content/chapter/LAN.png. ()  It has a permissions issue.  The source of the other graphics is https://s3.amazonaws.com/General_V88/boomyeah2015/codingdojo/curriculum/content/chapter/LAN.png and it is also access denied ().
  • Under Physical Network Topologies, I wish you wrote a little more about how home and office networks fit into these categories.  Because I am older, I happen to know that Ethernet (and that includes modern UTP and WiFi networks) are actually buses, even though they look like stars.  The bus in a modern LAN is actually inside the switch or even more importantly, in a hub.  The problems associated with the bus network topology is why the world has moved to switches, even in the home and office market.
  • Under  Wireless Topology, the acronym SP is seldom seen.  Usually, the term used is Internet Service Provider (ISP)
  • In the IPv6 IP Addresses, you write that there are enough IPv6 addresses to give every person on this planet 5*1028 addresses.  That should be 5*1028 addresses.  Let's see if the 28 in superscript makes it past the processing in your textbox.
  • The bullet point that IPv6 has no broadcasts is false at two levels: 1) Neighbor Discovery Protocol (NDP) uses broadcases and 2) The IPv6 address space includes definite multicast addresses:
    • 0xFF01 for interface local
    • 0xFF02 for link local.  "Link Local"  is anything that connects to the same port on the same router.  Link local will span switches and hubs
    • 0cFF04 is for admin local.  "admin local" has to be set up by the network admins
    • 0cFF05 is for site local.  "site local" has to be set up by the network admins
    • 0cFF08 is for organization local.  "Organization local" has to be set up by the network admins
    • 0cFF0e is for global.  I have no idea what a global address would mean in the real world.  I can see how a malicious player could mount a pretty good distributed denial of service attack.
  • In Protocols to Know, you include telnet.  Don't do that.  FTP should not be there, either.  You could put that these protocols are insecure and not used any more.  Use SSH (which includes SCP and rsh) instead.  This is a cyber-security class.  Also, the chart at the end of this page looks like it was clipped.  It certainly is incomplete.  
  • In Transport Layer, https://login.codingdojo.com/m/209/7725/53784, there is a protection error for the image.

Frameworks

What is a framework?  Judging from what I read, a framework is an outline of a thought process about computing security.  In many cases, I get the sense that they are different names for the same things, or at least very similar things.  For example, what is the difference between an SDN and a (for example) switch that can be configured with ACLs?

I wish there was a discussion about the different frameworks and advantages and disadvantages of each.  I am an IT manager: I want to choose a framework that will best allow me to accomplish my security goals at a reasonable cost.  Which framework do I pick, and why?

  • NIST the Cybersecurity Framework
  • CIC Controls
  • ISO/IEC 27002:2013   <++++ This might important on the test.
  • COBIT: an ISACA framework
  • ITIL

CIC Controls

In the Benchmarks and Secure configuration page, it mentions 20 CIS Controls.  CIS has reorganized the list and decreased the number of controls to 18.

Vendor diversity

What was written here is true.  But there is something important that was not written that is also true: Vendor diversity can be really hard to do.  Look how hard both Microsoft and Apple work to make their systems incompatible.  Microsoft Windows, for example, was POSIX compliant since Windows/NT 4.0, if not earlier!  Why does Mac OS X sometimes use a \r (carriage return) and sometimes a \n (newline)?  The keys to making vendor diversity work include, at least, 1) insistence that products rigorously enforce standards (be strict about what you send, liberal about what you accept), 2) insist that your staff document all the kludges, duct tape, and band-aids that were used to get two things to talk to each other, 3) allow time to figure these things out.

Causes of vulnerabilities

This is a comment on Vulnerabilities.

End of life systems.  While it is true that this is an issue with vendors, it is also an issue within our own organizations.  We have systems that we have made, and sometimes we don't know what they are or what they do or who uses them or are they even being used?

I had a job at google taking apart a software development lab that had (mostly) outlived its usefulness.  I turned off one computer and let it sit in a rack.  My plan was to let it sit in the rack for a week. If nobody screamed, then do an image backup and remove and dispose of the computer.  I got busy doing other things, and then on the 1st of the next month, somebody screamed.  There was a daemon on this server, which did something for about 5 minutes on the first of the month.  When the server was off, the daemon couldn't do whatever its client wanted it to do, so the client screamed for help.  The person who got the scream for help had no idea what it was that the client did, who ran it, or what they got out of it.  So we decided to do.... nothing... and see who screamed about that.  To my knowledge, nobody did.  On my second to the last day on that job, I documented everything I knew about this daemon and its client, got buy in that I had done the right thing in the right way, did the backup, documented where it could be found, and gave this to everybody on my team and to the team of people who ran the client.

How would somebody proactively check for something like this?

I suppose this goes under software inventory, which is one of the CIS controls.

Is there an automated way of thinking about this?

Run the following script:

# Foo Bar  foo1 foo12  foo12= Foo1 Foo12  Foo12=
for PW in foo bar  ; do
  # Column 1 is the LM hash, column 2 is the NTLM hash (but smbencrypt calls it the NT hash)
  echo $PW | xargs smbencrypt | awk '{print $2}' | tee looped_hash.txt
  echo "---------------------- $PW ------"
  date
  time /usr/sbin/john --show --format=NT looped_hash.txt
  done
